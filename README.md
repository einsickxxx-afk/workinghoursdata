# 해양사업부 근무실적 현황 대시보드

원본 근무시간 데이터(No.~비업무 중식 열)를 그대로 불러와 계산하는 정적 웹 대시보드입니다. 서버나 로그인 없이 파일만으로 동작합니다.

## 구성

- index.html : 대시보드 화면
- style.css : 디자인 (상단에 색상·폰트 변수 정리, 직접 수정해서 고정 가능)
- script.js : CSV 파싱, 통계 계산, 차트 렌더링 (외부 라이브러리 없이 동작)
- data/timesheet_sample.csv : 가상 샘플 데이터

## 데이터 형식

CSV 열 순서와 이름은 원본 그대로 사용합니다. 시간 값은 전부 분 단위 숫자입니다.

```
No., 구분, 년월, 주차, 사번, 이름, 직급, 사업장, 부서명,
월, 화, 수, 목, 금, 토, 일, 주확정근무시간, 평일확정근무시간, 법정휴게시간,
비업무_기타, 비업무, 비업무(중식)
```

- 년월: "202606" (숫자만, 하이픈 없음)
- 주차: "3주차(06월 15일~06월 21일)" 형식 (앞의 숫자+주차만 인식하므로 뒤 날짜 표기는 자유로움)
- 시간 열(월~일, 주확정근무시간 등): 전부 분 단위. 예) 552 = 9.2시간

자동으로 제외되는 대상

- 구분 = "근태예외자"
- 직급 = "상무" / "전무" / "부사장" / "전문위원"

열 이름이 하나라도 다르면 화면 상단에 어떤 열을 못 찾았는지 빨간 배너로 표시됩니다.

## 데이터 불러오기

화면 우측 상단 "CSV 불러오기" 버튼으로 파일을 직접 선택합니다. 인터넷 연결이나 로그인 없이 로컬에서 바로 반영되며, 인코딩(UTF-8/CP949/UTF-16)도 자동으로 인식합니다. xlsx 파일을 잘못 선택한 경우에도 바로 안내가 뜹니다.

GitHub Pages로 배포한 경우 data/timesheet_sample.csv를 실제 데이터로 덮어쓰고 커밋하면, 접속할 때마다 자동으로 그 데이터를 불러옵니다.

## 완성 파일로 저장 (매달 배포용)

1. CSV 불러오기로 그달 데이터를 불러옵니다.
2. "완성 파일로 저장" 버튼을 누르면, 그 데이터가 그대로 박힌 새 HTML 파일이 다운로드됩니다. (실제 CSV를 불러오지 않은 상태에서 누르면 경고가 뜹니다.)
3. 그 파일을 사내 공유 경로 등에 올리면, 다른 사람들은 CSV를 몰라도 열기만 하면 됩니다.

원본 파일 자체는 절대 스스로 바뀌지 않으며, 매번 새 파일이 별도로 만들어지는 방식입니다.

## 화면 구성

- KPI 카드 : 이번 주 52시간 근무 인원, 위험군, 평균 근무시간, 주말 근무 인원
- 주별 52시간 근무자 비율 추이 (최근 6주)
- 부서별 평균 주간 확정근무시간 (52시간 기준선 표시)
- 주 근무시간대별 인원 분포 (파이차트, 40시간 미만 ~ 52시간 이상 2시간 구간)
- 사업장별 평균 주간 확정근무시간
- 평일 요일별 근무시간 편차 (평일 평균 대비 위아래로 표시)
- 주말 근무 현황 (평일과 분리)
- 위험군 상세 표 (2주 이상 연속 52시간)

상단 필터(사업장/부서)를 바꾸면 모든 차트가 즉시 재계산됩니다. "이번 주"는 CSV에 담긴 주차 중 가장 최근 주차 기준입니다.

## 색상 · 폰트 설정

우측 상단 톱니바퀴 아이콘을 누르면 패널이 열립니다.

1. 색상 5개(주요 텍스트, 보조 텍스트, 차트 보조색, 연한 배경, 경고색)와 폰트를 원하는 대로 바꿉니다. 바꾸는 즉시 화면에 반영됩니다.
2. 마음에 드는 조합이 나오면 "style.css용 코드 복사"를 누릅니다. 코드가 화면에 표시되고 클립보드에도 복사됩니다.
3. style.css 최상단의 `:root{ ... }` 블록을 복사한 코드로 교체하면, 다음부터는 파일 자체에 색이 고정됩니다.

## GitHub Pages로 배포하기

1. 이 폴더 전체(파일 내용물)를 새 GitHub 저장소 최상단에 올립니다.
2. 저장소 Settings → Pages → Source에서 Deploy from a branch 선택, 브랜치는 main, 폴더는 / (root)로 저장합니다.
3. https://[아이디].github.io/[저장소명]/ 주소로 접속하면 됩니다.

## 로컬에서 미리 보기

파일을 더블클릭해서 열어도 "CSV 불러오기" 버튼으로 파일을 직접 선택하는 방식은 항상 동작합니다. data 폴더의 샘플이 자동으로 뜨는 것까지 보려면 아래처럼 간단한 로컬 서버를 띄우는 것을 권장합니다.

```
cd 대시보드_폴더
python3 -m http.server 8000
```

이후 브라우저에서 http://localhost:8000 으로 접속합니다. (파일을 그냥 더블클릭해서 열면 브라우저 보안 정책 때문에 샘플 CSV 자동 로딩만 안 될 뿐, CSV 불러오기 버튼은 문제없이 동작합니다.)
